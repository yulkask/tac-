Tac-Tickle — документация проекта
Этот документ описывает цель проекта, требования к игре «Так-Тикль» и правила
взаимодействия пользовательского интерфейса консольного приложения.
Файл обновлён для демонстрации последовательных локальных коммитов.
1. Общие сведения
Цель проекта. Разработать консольную игру в команде из трёх человек с соблюдением
требований к архитектуре, качеству кода, интерфейсу и процессу разработки.
Среда выполнения. Текстовый консольный интерфейс. Язык реализации — C#.
2. Обязательные требования
2.1 Функциональные требования
Игра работает в консоли и имеет текстовое меню.
Поддерживаются два игрока, которые ходят по очереди (компьютер не играет).
Реализованы:
сохранение и загрузка состояния игры (например, через меню);
запись результата завершённой партии в таблицу рекордов и вывод топ-N (5–10)
игроков при запуске;
проверка корректности пользовательского ввода (отсев невалидных ходов и
выход за границы);
пункты меню: начать новую игру, загрузить игру, показать рекорды, выйти.
Классы и методы разделены по ответственности (модули ввода-вывода, логики игры,
сохранений, рекордов).
2.2 Критерии оценки
1. Полное соответствие заданию и удобный интерфейс.
2. Соблюдение код-стайла и отсутствие предупреждений компилятора.
3. Наличие документации.
4. Корректное использование трёх уровней абстракции (пользовательский, бизнес, дата).
5. Использование в коде конструкторов, методов, параметров ( ref , o
ut , значения по
умолчанию), констант, свойств, перечислений, массивов, потоков/тасков/асинхронных
методов.
6. Работа в системе контроля версий: каждый участник выполняет ≥15 коммитов в
течение ≥2 месяцев.
3. Игра «Так-Тиκль»
3.1 Цель
Построить линию из трёх собственных фишек по горизонтали или вертикали (диагонали
разрешены как условие победы).
3.2 Материалы
2 игрока;
8 фишек (4 белые, 4 чёрные);
поле 4×4 с одноцветными клетками.
3.3 Правила
1. Каждый игрок получает 4 фишки одного цвета (белые начинают).
2. За ход можно переместить свою фишку на свободную клетку вверх/вниз/влево/вправо
(без диагоналей).
3. Стартовое расположение — фишки стоят через одну по краям поля.
4. Выигрывает тот, кто первым соберёт три фишки подряд по горизонтали, вертикали
или диагонали.
5. Если совершено 30 ходов и победителя нет, объявляется ничья.
3.4 Логика ввода
1. Игрок выбирает фишку по координате (например, A1 ).
2. Двигает её клавишами W,
A , S , D.
3. Подтверждает ход клавишей Enter.
3.5 Проверки
Фишка принадлежит текущему игроку.
Целевая клетка свободна.
Действие соответствует правилам.
На каждую ошибку выводится конкретное сообщение.
3.6 Стартовое поле
● — чёрные фишки, ○ — белые. Координатная сетка A–D / 1–4 совпадает с
изображением.

0 — выход
Правило ввода: при вводе числа вне диапазона 0–
4
4.2 Раздел «Начать новую игру»
Последовательность
1. Запрос имени игрока 1.
2. Запрос имени игрока 2.
3. Запуск игрового цикла.
4. Ход игрока 1.
5. Ход игрока 2.
6. Повтор шагов 4–5 до завершения партии.
Возможные события
отображается «Ошибка ввода».
Победа игрока 1 → «Ура, победа 1 игрока».
Победа игрока 2 → «Ура, победа 2 игрока».
Выход пользователя:
запрос «Сохранить игру?»;
«Да» → сохранить → выход в меню;
«Нет» → выход в меню.
После завершения
Запись результата в таблицу рекордов.
Возврат в главное меню.
На всех этапах доступна кнопка «выйти».
4.3 Раздел «Загрузить игру»
Если сохранений нет → сообщение «Игp пока не было» → меню.
Если сохранения есть → вывести список → пользователь выбирает игру.
После выбора:
загрузить состояние;
перейти в игровой процесс;
оставить кнопку «выйти», ведущую назад.
4.4 Раздел «Таблица рекордов»
Если записи есть → показать таблицу.
Если записей нет → сообщение «Рекордов пока нет».
Всегда доступна кнопка «выйти» → меню.
4.5 Раздел «Правила игры»
Показ текста правил.
Кнопка «выйти» возвратит к меню.
4.6 Раздел «Выход»
Завершение работы программы.

6. Техническое задание
6.1 Цель
Создать консольное приложение с меню, игровым процессом, сохранениями и таблицей
рекордов.
6.2 Функциональные требования
Главное меню
Отображает пункты: 1 — новая игра, 2 — загрузка, 3 — рекорды, 4 — правила, 5/0 —
выход.
При неверном выборе выводится сообщение об ошибке.
Начать новую игру
Запрашивает имена двух игроков.
Организует пошаговый процесс.
Позволяет выйти в любой момент.
При выходе спрашивает, сохранять ли игру ( Да
— сохранить,
Н
ет — отбросить).
При победе игрока выводит сообщение, записывает результат в таблицу и
возвращается в меню.
Загрузить игру
Хранит несколько сохранений.
При отсутствии сохранений сообщает «Игр пока не было».
При наличии выводит список и загружает выбранное состояние.
Во время процесса всегда можно вернуться назад.
Таблица рекордов
Содержит результаты завершённых игр.
Если таблица пустая, выводит соответствующее сообщение.
В разделе есть кнопка «выйти».
Правила игры
Показывает текст правил и кнопку «выйти».
Выход
Немедленно завершает программу.
6.3 Нефункциональные требования
Консольный интерфейс.
Простая навигация (в каждом экране есть опция «выйти»).
Сохранение данных в файл (подходящий формат: JSON, CSV)
